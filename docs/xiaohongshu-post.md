## 正文

家人们，有没有人跟我一样：

用 Claude Code / Cursor 疯狂 vibe coding，三天干出一个项目，爽是真的爽——
但某天想改个小功能，打开仓库一看：？？？这文件是谁写的？哦，是"我"写的。🙃

vibe coding 最大的痛不是写不出来，是**写完就失忆**：
- 所有上下文都埋在一条长长的 chat log 里，往上翻 300 条才能想起当时为啥这么改
- AI 唰唰改了 5 个文件，你只看到 diff，不知道它们在系统里长什么位置
- 项目越长越大，你脑子里那张"架构图"早就过期了

于是我索性（当然也是 vibe 出来的😂）写了个小工具：**CodeMap** 🗺️

一句话：**给你的代码库画一张实时地图，挂在副屏上。**

✨ 它长这样用起来：
- 一行命令启动：`node server.mjs /你的仓库`，浏览器打开就是一张力导向图，文件是节点，import 是连线
- AI 一改文件，对应的节点**当场亮起来**，还有个 "you are here" 📍 标记——你永远知道 AI 的手现在摸在系统的哪个角落
- 右边一条 session 时间线，这一局改过的文件按顺序排好，把线性的聊天记录变成空间上的"足迹"
- 最戳我的一点：接上 LLM 后，文件不按文件夹分组，而是按**它是干嘛的**分组——"用户认证""画布渲染""数据持久化"，直接是中文业务域气泡。代码看不懂没关系，地图是中文的🤝

而且它本身特别"轻"：**零依赖**，只要 Node 20+，整个项目就 950 行左右。不配 API key 也能用（退化成按文件夹分组），配上一个便宜模型（Haiku / 本地 Ollama 都行）就解锁语义分组，结果还会缓存，重启秒开。

用了几天最大的感受：vibe coding 的时候终于有了"上帝视角"，AI 在下面搬砖，我在地图上看它把哪个域搅动了。改完一轮，扫一眼时间线就知道这次 session 的"伤害范围"，心里有底多了。

---

🚧 当然，坦白局：它现在还是个很早期的 v0，坑不少——

1. import 分析目前只对 JS/TS 调教过，Python/Go 这些语言能出文件地图，但连线会很稀疏
2. 前端是 1 秒轮询 `/api/state`，仓库特别大的话性能还没优化过
3. 变更感知还不懂 git——只知道"这个文件动了"，不知道加了几行删了几行，点节点也还看不了 diff
4. README 里还躺着一个叫 Conductor 的大饼📐：让它反过来驱动 coding agent 分阶段干活、每阶段强制验收、还能打断点急停——目前只有设计文档，代码还没写

所以！这是个 MIT 开源项目，特别欢迎来玩 / 来提 issue / 来 PR：
加一门语言的 import 解析、把轮询换成 WebSocket、接 git diff、或者干脆来实现 Conductor，都是很好上手的切入点👏
